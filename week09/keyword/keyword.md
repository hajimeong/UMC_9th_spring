## 객체 그래프 탐색
### RDB (Relational DataBase)
관계형 데이터베이스의 약자로 데이터를 행과 열로 이루어진 테이블 형태로 저장하고,
여러 테이블 간 관계를 기반으로 데이터를 효율적으로 관리/조회할 수 있게 해주는 데이터베이스 모델

#### RDB(관계형 데이터베이스) 핵심 개념
1. 테이블 기반 구조
    - Excel처럼 표 형태로 구성된
    - 예: Users 테이블, Orders 테이블 등
2. 스키마 기반
    - 테이블의 컬럼, 타입, 제약조건 등을 정해진 구조에 맞춰야 함
    - 구조화된 데이터에 강함
3. 관계
    - 테이블 간 서로 연결되어 있음
    - `1:1`
    - `1:N`
    - `N:M`
4. SQL 사용
    - 데이터 조작 언어로 SQL(Structured Query Language) 사용
    - `SELECT`
    - `INSERT`
    - `UPDATE`
    - `DELETE`
5. 트랜잭션 보장
    - `ACID` 특성 제공
    - Atomicity(원자성)
    - Consistency (일관성)
    - Isolation (격리성)
    - Durability (지속성)

### 객체 그래프 탐색
객체는 자유롭게 객체 그래프를 탐색할 수 있어야 함
- 객체는 자유롭게 그래프를 탐색 가능, 각 상속, 연관 관계만 맺어져 있다면,</br> 멤버 -> 오더 -> 오더아이템 -> 아이템 -> 카테고리까지 접근 가능
- 하지만, DB의 경우 미리 그래프의 범위를 정해줘야 하고, 정해진 그래프 범위로 넘어갈 수 없으며, 데이터가 없으면 탐색할 수 없음

#### 비교
DB의 겨우 row로 데이터를 구분하는데, 이 데이터들은 서로 동일하지 않음 </br>
같은 row를 조회하더라도 ==연산자를 통해 비교 시 객체 관점에서는 서로 다른 인스턴스가 됨
